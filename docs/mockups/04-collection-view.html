Overview

This document outlines the high-level architecture for the API v2 migration. It covers the new endpoint structure, authentication flow, and rate limiting strategy.

Endpoint Structure

All v2 endpoints follow a consistent pattern with versioned response envelopes:

Method Path Description
GET/v2/resourcesList resources (paginated)
POST/v2/resourcesCreate resource
GET/v2/resources/:idGet single resource
PATCH/v2/resources/:idUpdate resource
DELETE/v2/resources/:idDelete resource

Rate Limiting

Rate limits are defined per API key tier. See the full specification in the ↗ API Specification · line 15 for exact limits per endpoint.

Default rate limit: 1000 requests per minute per API key. Burst allowance of 50 requests over 1 second. Headers included in every response:

  • X-RateLimit-Limit: Maximum requests per window
  • X-RateLimit-Remaining: Requests remaining
  • X-RateLimit-Reset: Unix timestamp of window reset
